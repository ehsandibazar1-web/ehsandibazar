# ساختارِ JSON برای «AI Import» مقاله

مرجعِ کاملِ فیلدهایی که فرمِ **AI Import** (`/adminpanel/ai-import`) از یک فایلِ JSON می‌خواند.
همه‌ی کلیدها اختیاری‌اند مگر جایی که «الزامی» نوشته شده. کلیدهای ناشناخته بی‌خطر نادیده گرفته می‌شوند.

## دو حالت

- **مقاله‌ی جدید:** کلیدِ `id` را نگذارید. یک مقاله‌ی تازه ساخته می‌شود.
- **ویرایشِ مقاله‌ی موجود (چرخه‌ی خروجی→ویرایش→ورودی):** کلیدِ `id` را با شناسه‌ی همان مقاله بگذارید.
  در این حالت **slug قفل است** و هر تغییری در آن نادیده گرفته می‌شود (خطِ قرمزِ URL).

## نمونه‌ی کامل

```json
{
  "id": 139,
  "lang": "fa",
  "title": "همه چیز در مورد کاراته",
  "slug": "همه-چیز-در-مورد-کاراته",
  "excerpt": "خلاصه‌ی کوتاهِ مقاله برای کارتِ فهرست.",
  "body": "<p dir=\"RTL\">متنِ کاملِ HTML با <a href=\"https://ehsandibazar.com/article/...\">لینک‌های داخلی</a> و <img src=\"...\"> و هر تگی...</p>",
  "seo_title": "همه چیز در مورد کاراته | تاریخچه و سبک‌ها",
  "meta_description": "توضیحاتِ متا برای گوگل (حداکثر ~160 کاراکتر).",
  "meta_keywords": "کاراته, هنر رزمی, دفاع شخصی",
  "canonical_url": null,
  "robots": null,
  "og_title": "همه چیز در مورد کاراته",
  "og_description": "توضیحِ اشتراک‌گذاری در شبکه‌های اجتماعی.",
  "image": "media/library/karate-hero.webp",
  "image_alt": "آشنایی با ورزش کاراته",
  "author_name": "احسان دیبازر",
  "reading_time": 17,
  "tags": ["کاراته", "دفاع شخصی"],
  "faqs": [
    { "question": "کاراته برای چه سنی مناسب است؟", "answer": "برای هر سنی..." }
  ],
  "status": "published",
  "published_at": "2024-01-01 10:00:00",
  "translation_of": null
}
```

## جدولِ فیلدها

| کلید | نامِ جایگزین (alias) | الزامی | توضیح |
|---|---|---|---|
| `id` | `article_id` | فقط برای ویرایش | شناسه‌ی مقاله‌ی موجود. نگذارید = مقاله‌ی جدید. |
| `title` | — | **بله** | عنوانِ مقاله. |
| `body` | `content` | **بله** | متنِ کاملِ HTML. **عیناً** ذخیره می‌شود (لینک‌های داخلی، عکس‌ها، `&zwnj;`/`&nbsp;` و کلِ تگ‌ها حفظ می‌شوند). |
| `slug` | — | خیر | آدرسِ صفحه. **فارسی مجاز و عیناً حفظ می‌شود** (فقط فاصله→خط‌تیره، کاراکترهای غیرمجاز حذف). نگذارید = از عنوان ساخته می‌شود. در حالتِ ویرایش **قفل** است. |
| `lang` | `locale` · `language` | خیر | `fa` (پیش‌فرض) یا `en`. |
| `excerpt` | — | خیر | خلاصه‌ی کارتِ فهرست. |
| `seo_title` | `seo.title` | خیر | عنوانِ سئو. |
| `meta_description` | `seo.meta_description` | خیر | توضیحاتِ متا. |
| `meta_keywords` | — | خیر | کلیدواژه‌ها (فعلاً فقط اطلاعاتی برای مقاله). |
| `canonical_url` | — | خیر | آدرسِ canonical. خالی = خودِ آدرسِ مقاله. |
| `robots` | — | خیر | مثلِ `index, follow` یا `noindex`. |
| `og_title` | `og.title` | خیر | عنوانِ اشتراک‌گذاری. خالی = `seo_title`. |
| `og_description` | `og.description` | خیر | توضیحِ اشتراک‌گذاری. خالی = `meta_description`. |
| `image` | `image_path` · `featured_image` · `hero_image` | خیر | **تصویرِ شاخص.** ببینید بخشِ «تصویرِ شاخص» پایین. |
| `image_alt` | — | خیر | متنِ جایگزینِ تصویر (ALT). |
| `author_name` | — | خیر | نامِ نویسنده. |
| `reading_time` | — | خیر | زمانِ مطالعه (دقیقه، عدد). |
| `tags` | — | خیر | آرایه‌ای از **نامِ** برچسب‌ها: `["کاراته","دفاع شخصی"]`. (فقط اضافه می‌کند، حذف نمی‌کند.) |
| `faqs` | `faq` | خیر | آرایه‌ای از `{ "question": "...", "answer": "..." }`. |
| `status` | `publish_status` | خیر | `draft` (پیش‌فرض) یا `published`. |
| `published_at` | `publish_date` | خیر | تاریخِ انتشار. |
| `translation_of` | — | خیر | شناسه‌ی مقاله‌ای که این ترجمه‌اش است. |
| `_content_hash` | — | خیر | از خروجیِ سیستم می‌آید؛ برای تشخیصِ تعارضِ نسخه. دست نزنید. |

## تصویرِ شاخص (`image`)

مقدارِ `image` یکی از این‌ها می‌تواند باشد و همه به مسیرِ دیسک تبدیل می‌شوند:

| ورودی | نتیجه |
|---|---|
| `media/library/foo.webp` (مسیرِ نسبی) | همان — بهترین حالت |
| `/storage/media/library/foo.webp` | `media/library/foo.webp` |
| `public/storage/media/library/foo.webp` | `media/library/foo.webp` |
| `https://ehsandibazar.com/public/storage/media/library/foo.webp` | `media/library/foo.webp` (بدونِ دانلود) |
| `https://external-site.com/x.jpg` (سایتِ دیگر) | دانلود می‌شود و مسیرِ نسبی ذخیره می‌گردد |

**نکته:** برای معرفیِ تصویری که **از قبل در کتابخانه‌ی رسانه** آپلود شده، ساده‌ترین راه دادنِ مسیرِ نسبیِ آن است (`media/library/…`). بعد از ایمپورت، هم فیلدِ «تصویرِ شاخص» در فرم پر می‌شود و هم هیرو روی خودِ سایت نمایش داده می‌شود.

## خطوطِ قرمز (هیچ‌وقت از فایل عوض نمی‌شوند در حالتِ ویرایش)

`slug` (URL) · وضعیتِ انتشار در صورتِ نبود · `user_id` · `lang` · `translation_of` · تعدادِ بازدید.
همچنین: هر فیلدی که در فایل **نباشد**، دست‌نخورده می‌ماند (چیزی پاک نمی‌شود).

## چه چیزی در چرخه نیست (از پنل انجام دهید)

- **دسته‌بندی (category):** از فرمِ ویرایشِ مقاله.
- تغییرِ ساختاریِ دیگر (مالک، زبان): از پنل.
